# 禁止项清单

> AI 绝对不可执行以下操作。违反任何一项必须立即停止并报告。

## 代码层面

### 绝对禁止

- **禁止** `eval()`、`Function()` 等动态代码执行
- **禁止** SQL 字符串拼接（必须用参数化查询）
- **禁止** 在代码中硬编码密钥、密码、Token
- **禁止** 关闭或绕过 TypeScript 严格模式（`// @ts-ignore`、`// @ts-nocheck`）
- **禁止** 使用 `any` 类型（确有需要时用 `unknown` + 类型守卫）
- **禁止** `console.log` 留在生产代码中（使用统一日志库）

### 危险操作

- **禁止** 未经确认执行 `DROP TABLE` / `TRUNCATE`
- **禁止** 直接操作生产数据库
- **禁止** 在不理解后果的情况下执行 `git push --force`
- **禁止** 删除或覆盖 `.env` 文件

## 架构层面

- **禁止** 跨层直接调用（如 Controller 直接调 Repository）
- **禁止** 在 Domain 层引入框架依赖
- **禁止** 模块间直接导入内部实现（必须通过公开接口）
- **禁止** 未经 ADR 进行架构级变更

## 文件层面

- **禁止** 修改 `AGENTS.md` 通用 AI 规范
- **禁止** 修改 `CLAUDE.md` 控制配置
- **禁止** 修改 `.claude/rules/` 目录下的文件
- **禁止** 修改 `.claude/guardrails/` 目录下的文件
- **禁止** 修改 `.claude/hooks/` 目录下的文件
- **禁止** 修改 `packages/*/AGENTS.md` 模块级上下文
- **禁止** 删除任何 `docs/` 下的规格文档
- **禁止** 在项目根目录随意创建文件

## 流程层面

- **禁止** 跳过测试直接提交
- **禁止** 需求不明确时自行假设并实现
- **禁止** 借修复 Bug 之名做大范围重构
- **禁止** 一次性提交超过 300 行的变更（应拆分为小步提交）

## 违规处理

如果 AI 意识到自己即将或已经违反禁止项：

1. **立即停止** 当前操作
2. **报告** 违反了哪条禁止项
3. **回滚** 已做的违规修改
4. **等待** 人工指导后再继续
