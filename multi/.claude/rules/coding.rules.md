# 编码规则

> 硬性规则。AI 在编写任何代码时必须遵守，无例外。

## 通用规则

1. **单一职责**: 每个函数/类只做一件事
2. **禁止硬编码**: 魔法数字和字符串必须提取为常量或配置
3. **类型安全**: 禁止使用 `any` 类型（TypeScript 项目），禁止绕过类型检查
4. **错误处理**: 禁止吞掉错误（空 catch），所有错误必须妥善处理或向上传播
5. **无副作用**: 纯函数优先，副作用必须显式隔离到边界层

## 安全规则

1. **禁止 SQL 拼接**: 必须使用参数化查询或 ORM
2. **禁止信任用户输入**: 所有外部输入必须校验和消毒
3. **禁止明文存储密码**: 必须使用 bcrypt/argon2 等哈希算法
4. **禁止在代码中硬编码密钥**: 密钥必须通过环境变量或密钥管理服务注入
5. **禁止暴露敏感信息**: 错误响应和日志中不得包含密码、Token 等

## 代码风格

1. **函数长度**: 单个函数不超过 50 行，超过则拆分
2. **文件长度**: 单个文件不超过 300 行，超过则拆分模块
3. **嵌套深度**: 最多 3 层嵌套，超过则提前返回或提取函数
4. **命名规范**: 详见 `.claude/guardrails/naming.md`
5. **注释**: 只在逻辑不自明时添加注释，解释"为什么"而非"是什么"

## 依赖管理

1. **最小依赖**: 能用标准库解决的不引入第三方包
2. **锁定版本**: 依赖版本必须精确锁定，禁止使用 `^` 或 `~`
3. **新增依赖审批**: 引入新依赖前必须说明理由和替代方案
