# 通用任务执行协议

> 适用于所有类型的任务。特定类型（功能/缺陷）有各自的扩展协议。

## 阶段一: 理解

1. **阅读任务分配**: 在 `docs/tasks/CURRENT.md` 找到自己的当前任务，在 `docs/tasks/BOARD.md` 查看任务详情
2. **确认范围**: 明确哪些文件可以修改、哪些不可以
3. **识别依赖**: 确定任务涉及的模块和它们的依赖关系
4. **检查记忆**: 查阅 `.claude/memory/` 是否有相关的已知问题或模式
5. **提出疑问**: 如有任何不明确的地方，先提问再动手

## 阶段二: 规划

1. **拆解步骤**: 将任务拆解为可独立完成的小步骤
2. **确定顺序**: 按依赖关系排列执行顺序
3. **预判风险**: 识别可能的阻塞点和风险
4. **陈述计划**: 向用户简要说明执行计划，确认后再开始

## 阶段三: 执行

1. **小步前进**: 每次只改一个小点，保持可回退
2. **持续验证**: 每个步骤完成后运行相关测试
3. **遵守规则**: 严格遵循 `.claude/rules/` 中的所有规则
4. **不越界**: 只做任务要求的事，不做额外的"改进"

## 阶段四: 验证

1. **运行测试**: 确保所有测试通过
2. **自审代码**: 按 `.claude/rules/review.rules.md` 进行自审
3. **检查范围**: 确认没有修改任务范围外的文件

## 阶段五: 收尾

1. **更新进度**: 更新 `docs/tasks/BOARD.md` 任务状态和 `docs/tasks/CURRENT.md` 索引
2. **更新记忆**: 如有新发现，更新 `.claude/memory/`
3. **总结交付**: 向用户简要说明完成情况和注意事项
