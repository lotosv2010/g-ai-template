# Shared 通用模块 - AI 上下文

> 本文件定义 `shared` 包的模块级 AI 开发上下文。
> AI 在修改本模块代码前必须阅读。

## 模块职责

**共享基础库** - 提供跨包复用的工具函数、类型定义和常量。

- 通用 TypeScript 类型和接口
- 全局常量和枚举定义
- 工具函数（日期、字符串、数据转换等）
- 通用校验规则（前后端共用）

## 内部架构

```
src/
├── types/            # 共享类型定义
│   ├── api.ts            # API 请求/响应通用类型
│   ├── [domain].ts       # 按业务领域分组的类型
│   └── index.ts
│
├── constants/        # 全局常量和枚举
│   ├── error-codes.ts    # 错误码定义
│   ├── [domain].ts       # 按领域分组的常量
│   └── index.ts
│
├── utils/            # 工具函数
│   ├── date.ts
│   ├── string.ts
│   ├── [category].ts
│   └── index.ts
│
├── validators/       # 通用校验规则
│   ├── [domain].ts       # 按领域分组的校验
│   └── index.ts
│
└── index.ts          # 统一导出入口
```

## 依赖规则

### 包间依赖

| 可以依赖 | 禁止依赖 |
|----------|----------|
| 无（零业务包依赖） | `web`、`server`、`ai` 或任何业务包 |

### 被依赖方

所有其他包都依赖 shared，因此：

> **修改本包影响面最大，必须格外谨慎。**

## 变更规则

1. 导出 API 必须向后兼容，破坏性变更需通知所有依赖方
2. 所有导出的函数/类型必须有单元测试
3. 新增外部依赖需说明理由（优先用原生 API）

## 本模块特有约定

- 所有对外内容必须通过 `src/index.ts` 统一导出
- 禁止其他包引用 shared 内部路径（如 `@scope/shared/src/utils/xxx`）
- 工具函数必须是纯函数（无副作用）
- 类型定义按业务领域分文件，不堆在一个文件中
- 常量使用 `UPPER_SNAKE_CASE` 并注释说明用途
