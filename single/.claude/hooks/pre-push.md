# Hook: 推送前检查

> 在代码推送到远程仓库前自动执行的检查。
> 比 pre-commit 更严格，确保推送到远程的代码质量。

## 触发时机

- `git push` 执行前

## 检查项

### 1. 完整测试

```bash
# 运行全量测试
[例: pnpm test]
```

### 2. 构建检查

```bash
# 确保项目能正常构建
[例: pnpm build]
```

### 3. 分支保护

- 禁止直接推送到 `main` / `master` 分支
- 推送前确认目标分支正确

### 4. 文档同步检查

- 如有 API 变更，检查 `docs/API_SPEC.md` 是否已更新
- 如有模型变更，检查 `docs/DATA_MODEL.md` 是否已更新

## 失败处理

检查失败时：
1. 显示失败原因
2. 阻止推送
3. 提供修复建议
