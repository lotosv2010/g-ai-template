# 技能: 编写测试

> 当需要为功能编写测试时，使用此技能模板。

## 测试分层

```
tests/
├── unit/              # 单元测试 - 测试单个函数/类
│   └── [module]/
├── integration/       # 集成测试 - 测试模块间协作
│   └── [module]/
└── e2e/               # 端到端测试 - 测试完整流程
    └── [scenario]/
```

## 单元测试规范

### 命名规则

```
describe('[被测模块/函数名]', () => {
  it('当 [条件] 时，应该 [预期结果]', () => {})
})
```

### 结构: AAA 模式

```
// Arrange - 准备测试数据和环境
// Act - 执行被测行为
// Assert - 验证结果
```

### 要求

- 每个测试只验证一个行为
- 测试之间互相独立，不共享可变状态
- 使用 Mock 隔离外部依赖
- 覆盖: 正常路径 + 边界情况 + 错误路径

## 集成测试规范

### 要求

- 使用测试数据库（不使用生产数据）
- 每个测试前后清理数据
- 测试真实的模块间交互
- 重点测试数据流经多层后的正确性

## 测试场景清单

每个功能至少覆盖:

- [ ] 正常输入 -> 正常输出（Happy Path）
- [ ] 缺少必填字段 -> 校验错误
- [ ] 无效数据格式 -> 校验错误
- [ ] 无权限访问 -> 403 错误
- [ ] 资源不存在 -> 404 错误
- [ ] 并发操作 -> 数据一致（如适用）
- [ ] 边界值 -> 正确处理（空列表、最大值等）

## 禁止行为

- 禁止测试中使用 `sleep` / 固定等待
- 禁止测试依赖外部网络服务
- 禁止测试之间有执行顺序依赖
- 禁止在测试中使用生产数据
